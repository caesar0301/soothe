# Draft RFCs

This directory contains draft RFC specifications that are works-in-progress.

## Purpose

The drafts directory is for:

- Initial design exploration
- Work-in-progress RFCs
- RFCs not yet ready for formal review
- Experimental design ideas

## Lifecycle

RFCs typically move through this lifecycle:

1. **Draft** (here) → Initial design work
2. **Proposed** → Moved to `../specs/` for review
3. **Accepted** → Ready for implementation
4. **Implemented** → Fully implemented in code

## Moving from Draft to Proposed

When a draft RFC is ready for review:

1. Move it from `drafts/` to `../specs/`
2. Update status from "Draft" to "Proposed"
3. Add to `rfc-history.md` and `rfc-index.md`
4. Use `platonic-specs` to validate
