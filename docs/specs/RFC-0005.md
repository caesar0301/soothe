# RFC-0005: Weaver Agent Architecture Design

**RFC**: 0005
**Title**: Weaver Agent Architecture Design
**Status**: Draft
**Created**: 2026-03-13
**Related**: RFC-0001, RFC-0002, RFC-0003, RFC-0004

## Abstract

This RFC defines the architecture of the Weaver agent, a generative subagent framework that composes skills, tools, and MCP capabilities into instant Soothe-compatible subagents. Weaver can reuse previously generated agents when fit is sufficient, or synthesize a new agent package with bounded scope and policy controls.

## Motivation

Soothe currently provides fixed bundled subagents. Users also need dynamic, task-specific subagents that can be created at runtime from existing ecosystem building blocks without manually scaffolding code. Weaver provides this adaptive generation layer while preserving Soothe protocol guarantees.

## Design Principles

### Compose, do not reinvent

Weaver prioritizes composition of Skillify outputs, LangChain tools, MCP adapters, and deepagents subagent wrappers rather than bespoke runtime engines.

### Policy-gated agent generation

Generated capabilities are subject to policy checks for tool/MCP access, data handling, and publication before registration.

### Deterministic packaging + reuse first

Before generating, Weaver searches an index of generated agents and reuses a suitable existing package when possible.

## Component Overview

### AgentRequirementAnalyzer

- Converts user objective into required capabilities, constraints, and acceptance criteria
- Produces a generation plan and reuse query

### AgentReuseIndexer

- Tracks generated agents, summaries, embeddings, and capability tags
- Returns candidate reusable agents with confidence score

### AgentComposer

- Resolves required skills via Skillify retrieval
- Resolves tool and MCP dependencies from configured registries
- Produces a generation blueprint (prompt + scaffold parameters)

### SubagentGenerator

- Executes bundled `create-subagent` skill to synthesize a Soothe-compatible subagent package
- Ensures generated artifact conforms to expected folder layout and entry points

### GeneratedAgentRegistry

- Stores metadata, versions, and workspace paths for generated agents
- Publishes selected agents to Soothe routing layer

## Data Flow

1. Receive user request for specialized capability
2. Analyze requirements and derive capability signature
3. Query reuse index for existing generated agents
4. If reusable candidate meets threshold, return/load candidate
5. Otherwise resolve skills (via Skillify), tools, MCP connections
6. Run generator workflow using `create-subagent` skill
7. Validate package + policy gates
8. Register and index new generated agent
9. Route execution to generated agent or return registration result

## Architectural Constraints

- Weaver MUST run as a Soothe-compatible subagent type.
- Generated agents MUST conform to Soothe loading and routing expectations.
- Tool selection MUST reuse langchain tools (`BaseTool`/`@tool`) where available.
- MCP connections MUST use `langchain-mcp-adapters`.
- Every generation/publish action MUST be checked through `PolicyProtocol`.
- Generated-agent registry and index MUST be durable and resumable.
