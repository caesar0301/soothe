# RFC-0004: Skillify Agent Architecture Design

**RFC**: 0004
**Title**: Skillify Agent Architecture Design
**Status**: Draft
**Created**: 2026-03-13
**Related**: RFC-0001, RFC-0002, RFC-0003

## Abstract

This RFC defines the architecture of the Skillify agent, a Soothe-compatible subagent responsible for skill lifecycle management: discovery, normalization, storage, indexing, and semantic retrieval. Skillify is designed as a durable long-running capability that can both continuously curate a skill warehouse and serve on-demand skill packs for user goals.

## Motivation

The current Soothe stack can use skills through deepagents `SkillsMiddleware`, but it lacks a protocolized subsystem that continuously acquires and curates reusable skills from diverse sources. A dedicated Skillify architecture enables:

- Reusable skill curation across sessions and threads
- Semantic retrieval of relevant skills for downstream planning and execution
- Durable provenance and indexing of skill artifacts
- Strict policy-governed acquisition and publication workflows

## Design Principles

### Protocol-first skill lifecycle

Skillify integrates with Soothe protocols (`PolicyProtocol`, `DurabilityProtocol`, `VectorStoreProtocol`, `ContextProtocol`) and avoids runtime-tight coupling so implementations remain swappable via `SootheConfig`.

### Reuse deepagents and langchain ecosystem

Skillify delegates browsing and extraction to existing subagents/tools and relies on deepagents skill format (`SKILL.md`) for compatibility.

### Durable, auditable curation

All fetched and generated skills include source metadata and lifecycle state so skill lineage can be audited and recovered.

## Component Overview

Skillify has two operating modes:

1. **Background curation loop**: discover, parse, normalize, validate, index, and store skills.
2. **Interactive retrieval service**: retrieve and return best-fit skill bundles for an incoming user objective.

### Skillify Runtime Components

#### SkillSourceAdapter

- Collects candidate skills from configured sources (web pages, git repos, local catalogs)
- Emits normalized source documents with provenance metadata

#### SkillNormalizer

- Converts source documents to deepagents-compatible skill bundles
- Enforces required structure (`SKILL.md`, optional assets/scripts/references)

#### SkillRegistry

- Maintains durable warehouse metadata and lifecycle states (`candidate`, `validated`, `published`, `deprecated`)
- Persists canonical storage locations under the Skillify workspace

#### SkillIndexer

- Generates embeddings and indexes skill summaries in configured Soothe vector store backend
- Supports semantic and metadata-filtered retrieval

#### SkillRetrievalService

- Accepts an objective and returns ranked skill bundles and rationale
- Applies policy checks before exposing external or restricted skills

## Data Flow

### Curation Flow

1. Schedule/discover source candidates
2. Check acquisition action with `PolicyProtocol`
3. Delegate extraction to subagents/tools (e.g., browser)
4. Normalize into deepagents skill structure
5. Validate package quality and metadata completeness
6. Persist warehouse artifact and registry record
7. Embed and index searchable projections
8. Emit `soothe.skillify.*` telemetry events

### Retrieval Flow

1. Receive objective/query
2. Project query context
3. Search vector index + metadata filters
4. Rank candidates by semantic fit + policy constraints
5. Return bounded skill bundle list and provenance

## Filesystem and Storage Layout

Skillify storage follows Soothe agent workspace conventions:

```bash
~/.soothe/agents/skillify/
  warehouse/                # canonical skill artifacts
  index/                    # optional local index cache/manifest
  runs/                     # curation run logs and checkpoints
  tmp/                      # delegated subagent temp data
```

Skill artifacts remain compatible with deepagents `SkillsMiddleware` discovery rules.

## Architectural Constraints

- Skillify MUST run as a Soothe subagent (`SubAgent` or `CompiledSubAgent`).
- Skill packages MUST be deepagents-compatible (`SKILL.md` contract).
- All external acquisition and publication actions MUST pass `PolicyProtocol`.
- Indexing MUST use Soothe vector store abstractions (`VectorStoreProtocol`).
- Runtime state MUST be recoverable through `DurabilityProtocol`.
- Retrieval responses SHOULD degrade gracefully (partial results > hard fail).

## Configuration Surface (SootheConfig Extensions)

Recommended configuration section:

- `skillify.enabled`
- `skillify.workspace_dir`
- `skillify.curation.enabled`
- `skillify.curation.interval_seconds`
- `skillify.sources[]`
- `skillify.index.top_k_default`
- `skillify.validation.min_quality_score`

## Observability

Skillify should emit custom stream events in canonical format:

- `soothe.skillify.curation.started`
- `soothe.skillify.curation.item_ingested`
- `soothe.skillify.curation.item_rejected`
- `soothe.skillify.index.updated`
- `soothe.skillify.retrieve.completed`
